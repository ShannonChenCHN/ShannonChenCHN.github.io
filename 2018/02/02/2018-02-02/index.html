Feb 2, 2018

最近这段时间比较充裕，自己一边复习一边整理些笔记，感觉收获不少。其实最好的节奏是一边实践，一边系统学习。有输入，有思考消化，有输出，这是最理想的学习模型了。（想起去年这个时候，OnlyU 第一版刚上线，忙的要死，所以要好好珍惜现在所拥有的呀）

在复习总结的过程中，确实感觉到比较累，所以在做的时候，甚至会怀疑自己的决定和行为是否是对的。

所以我们要明白自己现在处于哪个阶段，需要做的是什么，如果现在不去做该做的事情，难道等到三年后五年后十年后来做？等到那个时候才意识到自己本该在入行前两年要做的，却没做好，再去补的话，就晚了。因为那个时候有太多其他要紧的事去做。比如说我们在入行三年内要把 iOS 开发的基础、中级内容都掌握，还要有自己擅长的领域，要想实现这个阶段性的目标，那就最好早点去落实、完成，而不是等到三五年后才想起来自己原来缺了这么多，最后不了了之，成了“差不多先生”。

昨天看到一篇公众号文章提到了两个要点，一个是技术的三个层次，另一个是技术精进如何找标杆。

技术的三个层次：

  • 第一阶段：基于 API 开发（其实就是指会用）
  • 第二阶段：阅读源码，洞悉内在原理（跟我最近在看别人的源码和书、文档一样，还是在学习吸收状态）
  • 第三阶段：优化框架（知道各个框架、工具、技术的优缺点，会有针对性的优化，能够自己造出适合的轮子）

技术精进如何找标杆：

  • 优秀的人
  • 优秀的作品
  • 一般性规律
  • 技术本身的知识层次
  • 项目指标

前段时间把自己开源框架 YHListKit 的文档放到掘金上了，本来之前审核没通过，后来也就没再管它。但是最近这两天却又被推荐了，两天之内收到了 30 多个赞，50 个左右的新关注。相比简书（流量主要来自搜索、向专题投稿、关注订阅）、公众号（基本上靠推荐、关注订阅）等平台，掘金的推荐系统更利于获取关注、获取流量。
另一方面，因为这篇文章是介绍自己的开源项目，这个开源项目也是花了不少精力才最终出炉的，而且这个轮子所涉及的应用方向是大家平时基本上都会碰到的，所以才会这么么容易受关注。

总结下来，作为技术自媒体，能够吸引关注的几个要点：

  • 经过思考和实践的产出，有自己的独特性，而不是模仿、记笔记之类的
  • 解决痛点，对别人有帮助
  • 所关注的领域受众广（比如你写 Swift 服务端的话，大部分 iOS 开发者都不怎么关心）
  • 选择合适的平台

今天早上看了一些 Swift 服务端的框架，突然对服务端很感兴趣，很想找个时间学学服务端的技术。

解决拖延症的一个好办法，就是按优先级列出所有要做的事情，优先把最重要的先做了。比如说每天晚上要写写日记，那就最好在下班一回来就写。

今天无意间看了下 isaced 的博客，发现他的知识面很广，除了 iOS 开发，还会玩些 Python、Swift 服务端、Mac 应用开发、Jenkins 搭建等等。他是阿里的工程师，所以我想阿里的工程师，应该在技术上会有一定的深度、广度，还会玩些新的技术、工具，更重要的是“玩”的心态。
