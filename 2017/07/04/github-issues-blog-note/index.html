什么， GitHub Issues 还可以用来记笔记、写博客！？

Jul 4, 2017

“你在写什么？”
“随便记点笔记”
“为什么不用印象笔记，而用这个记呢？”
“这个啊。。。。。。我先喝口水哈”
“。。。。。。”

今天有朋友看到我在记笔记📒，然后就出现了上面的对话。我以前一直是用印象笔记来记笔记的，感觉也蛮不错，但是今年改用 『GitHub Issues』 来记笔记了。

等等，GitHub Issues？这是什么鬼？有些不是程序员的朋友可能觉得有些陌生。接下来我们就聊聊 GitHub Issues 吧：

  • GitHub Issues 是什么
  • GitHub Issues 和其他记笔记方式的对比
  • 为甚么有人会用 GitHub Issues 写博客

1. GitHub Issues 是什么

GitHub 是一个用来托管代码的地方，GitHub 上的每个项目/代码仓库都有一个 GitHub Issues 的功能模块，这个 GitHub Issues 就是用来对该项目进行追踪任务、bug 和改进项的。

2. GitHub Issues 和其他记笔记方式的对比

注：这几个对比的点主要以我自己的需求的为参照。

产品 支持 Markdown 使用体验 支持分享

GitHub Issues| ✅|界面简洁，排版清晰，可以加 tag，可以跟 GitHub 项目结合一起用|可以直接将网页链接分享出去，可以随时看到更新的内容，还可以在下面评论|
印象笔记|❌|支持多种功能和文件，格式不太好调|支持群聊、分享到邮件|
有道云笔记|✅|没用过，不做评价|可以直接将网页链接分享出去，可以随时看到更新的内容|
锤子便签|✅|界面简洁|可以转成图片分享出去|

综合，比较起来，我更喜欢 GitHub 的简洁，编辑方便，分享方便，而且还可以打 tag。

3. 为甚么有人会用 GitHub Issues 写博客

我自己开始用 GitHub Issues 来记录东西是受 @phodal 大神的启发，觉得确实是个用来自我管理和记录的不错的方式。

用玉伯的话来说，博客的本质是记录，是与自己、他人的内心对话。
所以 GitHub Issues 完全可以当成博客来写，别人能看到，自己也能看到。

从实际角度上来看，一不用花钱买服务器自己去折腾一个博客，二没有那么重的仪式感，写的内容可正式可随意也可当成笔记来写。
